# VuePress

# 目录结构

结构如下

projcet
├── docs
│   ├── .vuepress (可选的)
│   │   ├── components (可选的)
│   │   ├── theme (可选的)
│   │   │   └── Layout.vue
│   │   ├── public (可选的)
│   │   ├── styles (可选的)
│   │   │   ├── index.styl
│   │   │   └── palette.styl
│   │   ├── templates (可选的, 谨慎配置)
│   │   │   ├── dev.html
│   │   │   └── ssr.html
│   │   ├── config.js (可选的)
│   │   └── enhanceApp.js (可选的)
│   │ 
│   ├── README.md
│   ├── guide
│   │   └── README.md
│   └── config.md
│
└── package.json

注意

请留意目录名的大写。

  • docs/.vuepress: 用于存放全局的配置、组件、静态资源等。
  • docs/.vuepress/components: 该目录中的 Vue 组件将会被自动注册为全局组件。
  • docs/.vuepress/theme: 用于存放本地主题。
  • docs/.vuepress/styles: 用于存放样式相关的文件。
  • docs/.vuepress/styles/index.styl: 将会被自动应用的全局样式文件，会生成在最终的 CSS 文件结尾，具有比默认样式更高的优先级。
  • docs/.vuepress/styles/palette.styl: 用于重写默认颜色常量，或者设置新的 stylus 颜色常量。
  • docs/.vuepress/public: 静态资源目录。
  • docs/.vuepress/templates: 存储 HTML 模板文件。
  • docs/.vuepress/templates/dev.html: 用于开发环境的 HTML 模板文件。
  • docs/.vuepress/templates/ssr.html: 构建时基于 Vue SSR 的 HTML 模板文件。
  • docs/.vuepress/config.js: 配置文件的入口文件，也可以是 YML 或 toml。
  • docs/.vuepress/enhanceApp.js: 客户端应用的增强。

注意

当你想要去自定义 templates/ssr.html 或 templates/dev.html 时，最好基于 默认的模板文件 来修改，否则可能会导致构建出错。

对于上述的目录结构，默认页面路由地址如下：

文件的相对路径 页面路由地址
/README.md /
/guide/README.md /guide/
/config.md /config.html

# 环境搭建

# 安装软件

  • VSCode
  • Node.js
  • Yarn 或 npm（包管理工具，推荐Yarn）

注意

请确保你的 Node.js 版本 >= 8

# 搭建步骤

# 全局安装 vuepress
yarn global add vuepress # 或者：npm install -g vuepress

# 新建项目和进入项目
mkdir project && cd project

# 初始化项目
yarn init -y # 或者 npm init -y

# 新建docs文件夹
mkdir docs

# 在docs目录中，创建.vuepress目录
mkdir .vuepress

# 在.vuepress目录中，创建public目录
mkdir public

# 在.vuepress目录中，创建config.js文件
touch config.js

# 将 VuePress 作为一个本地依赖安装，请在项目文件路径执行（装插件要用）
yarn add -D vuepress # 或者：npm install -D vuepress

这时候，你项目结构应该是这样。

project
├── docs
│   ├── .vuepress
│   │   ├── public
│   │   └── config.js
│   │
│   └── README.md
│
└── package.json

# 配置config.js

module.exports = {
  title: '个人主页', 
  description: '博客',
  head: [
    ['link', { rel: 'icon', href: '/img/logo.ico' }],
    ['link', { rel: 'manifest', href: '/manifest.json' }],
  ]
}

# 配置package.json

此处我们把 docs 目录作为 targetDir(目录) ，下面所有的“文件的相对路径”都是相对于 docs 目录的。在项目根目录下的 package.json 中添加 scripts(脚本) ：

{
  "scripts": {
    "docs:dev": "vuepress dev docs",
    "docs:build": "vuepress build docs"
  }
}
如何使用 
# 要启动本地服务，运行
$ yarn docs:dev # 或者：npm run docs:dev
# 要生成静态的 HTML 文件，运行：
$ yarn docs:build # 或者：npm run docs:build

默认情况下，静态 HTML 文件将会被生成在 `.vuepress/dist` ，
当然，你也可以通过 `.vuepress/config.js` 中的 `dest` 字段来修改，
生成的文件可以部署到任意的静态文件服务器上，参考 部署 来了解更多。

提示

VuePress中有两个命令，vuepress dev docs 命令运行本地服务，通过访问 localhost:8080 来预览网站效果，vuepress build docs 命令用来生成静态文件，在这里将两个命令封装成脚本的方式。

配置完这些就可以启动本地服务了。

# 基本配置

一个 VuePress 网站必要的配置文件是 .vuepress/config.js，它应该导出一个 JavaScript 对象, 主要配置包括网站的标题、描述等基本信息，以及主题的配置。这里列举一下常用配置。

# 网站信息

module.exports = {
    title: '个人主页', 
    description: '庞希的博客',
    head: [
        ['link', { rel: 'icon', href: '/img/logo.ico' }],
        ['link', { rel: 'manifest', href: '/manifest.json' }],
    ]
}
  • title：网站标题
  • description：网站描述
  • 额外需要被注入到当前页面的HTML"head"中的标签，其中路径的"/"就是public资源目录。

# 主题配置

module.exports = {
    themeConfig: {
    nav: [
        { text: '主页', link: '/' },
        { text: '博文',
        items: [
            { text: '我', link: '/i/' },
            { text: '你', link: '/you/' },
            { text: '他', link: '/he/' }
            ] 
            },
            { text: '关于', link: '/about/' },
            { text: 'Github', link: 'https://www.github.com/codeteenager' },
        ],
    sidebar: {

        '/me/': [
            'a',
            'b',
            ],
        '/you/': [

            'a',
            'b',
            ],
        '/he/': [
            'a',
            'b',
            ]
    },

    sidebarDepth: 2,
    lastUpdated: '最后更新',
}

  • nav：导航栏（网页的上方），主要用于配置导航栏的链接，例如以上主页的link为"/"，默认是根目录下的README.md。"/android/"就是 project → docs → android → README.md 文件。

  • sidebar：侧边栏，此配置分为三种：自动、数组、对象。例子使用的是对象。后面a、b都是md文件，可以省略.md拓展名。开头是对象路径以"/"结尾的路径将会被视为 */README.md。"/android/"是指根目录下android文件夹中的路由，每个路由链接都要有README.md。所以目录结构如下：

├── docs
├── README.md
├── i
│   └── README.md
├── you
│   └── README.md
└── he
    └── README.md

  • sidebarDepth：嵌套的标题链接深度，默认的深度为1，可选0，1，2。

  • lastUpdated：时间戳，git跟踪时间，可装插件改时间的格式

# PWA配置

VuePress默认支持PWA配置的，需要在基本配置中开启serviceWorker。

module.exports = {
     serviceWorker: true,
}

然后再添加icons和Manifest配置，在public中添加manifest.json配置，和图标。如果不知道PWA的可以到PWA配置查看相关资料。

{
  "name": "姜帅杰",
  "short_name": "姜帅杰",
  "start_url": "index.html",
  "display": "standalone",
  "background_color": "#2196f3",
  "description": "姜帅杰的个人主页",
  "theme_color": "blue",
  "icons": [
    {
      "src": "./logo.png",
      "sizes": "144x144",
      "type": "image/png"
    }
  ],
  "related_applications": [
    {
      "platform": "web"
    },
    {
      "platform": "play",
      "url": "https://play.google.com/store/apps/details?id=cheeaun.hackerweb"
    }
  ]
}

在config.js配置中添加manifest.json，由于iphone11.3不支持manifest的图标，所以加上apple-touch-icon图标配置即可。

module.exports = {
    head: [
            ['link', { rel: 'manifest', href: '/manifest.json' }],
            ['link', { rel: 'apple-touch-icon', href: '/img/logo.png' }],
          ]
}

最后在iphone中访问网站，然后添加主屏幕即可。

# 自定义页面

默认的主题提供了一个首页（Homepage）的布局(用于这个网站的主页)。想要使用它，需要在你的根级 README.md添加数据。

---
home: true
heroImage: /hero.png
actionText: 快速上手 →
actionLink: /zh/guide/
features:
- title: 简洁至上
  details: 以 Markdown 为中心的项目结构，以最少的配置帮助你专注于写作。
- title: Vue驱动
  details: 享受 Vue + webpack 的开发体验，在 Markdown 中使用 Vue 组件，同时可以使用 Vue 来开发自定义主题。
- title: 高性能
  details: VuePress 为每个页面预渲染生成静态的 HTML，同时在页面被加载的时候，将作为 SPA 运行。
footer: MIT Licensed | Copyright © 2018-present Evan You
---

如果你想自定义首页或者其他页面，可以在页面的md文件中添加页面Vue文件。Vue文件放置在 docs/.vuepress/components 目录中。

---
layout: HomeLayout
---

# 部署

由于构建的时候生成静态页面，我们将 dist 文件夹中的内容部署在 Gitee、GitHub等 的 Pages。如果使用 git 上传，操作比较繁琐，这里使用脚本的方式部署。

# 配置SSH

  1. 执行git config命令预先配置好相关的用户信息
$ git config --global user.name "你的名字或昵称"
$ git config --global user.email "你的邮箱"

# 该命令可看到自己的配置
$ git config --list
  1. 生成/添加SSH公钥
# 生成公钥
$ ssh-keygen -t rsa -C "你的邮箱" # 输入完一直按回车就行

# 查看公钥
$ cat ~/.ssh/id_rsa.pub # 此时公钥出现，可复制
  1. 在 https://gitee.com/profile/sshkeys 添加公钥

  1. 验证连接
$ ssh -T git@gitee.com # 可能需要输入Yes

若返回 Hi XXX! You've successfully authenticated, but Gitee.com does not provide shell access. 内容，则证明添加成功。

# 创建和配置deploy.sh

提示

以下有说明部署到各个平台

# 在project下创建deploy.sh
$ touch deploy.sh
deploy.sh 脚本内容，复制粘贴改用户名即可。 
#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
set -e

# 生成静态文件
yarn docs:build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.example.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果发布到 https://<USERNAME>.gitee.io/<REPO>，把下面#号去掉，改成自己的信息。
# git push -f git@gitee.com:<USERNAME>/<REPO>.git master:gh-pages
cd -

# 执行deploy.sh脚本

在根目录 package.json 文件添加一条代码后保存

{
  "scripts": {
    "deploy": "bash deploy.sh"
    },
}

用该命令执行脚本，即可自动构建并部署到 Gitee 上

$ yarn deploy # 或者 npm run deploy

# 注意事项

  1. 新建的仓库语言选择HTML
  2. 脚本执行原理，是将生成静态文件目录 docs/.vuepress/dist 推送到 gh-pages 分支

# 自定义域名

# Gitee Pages Pro 自定义域名

# 前期工作

  1. 开启pages服务，在没有使用自定义域名之前，你的 user-name.gitee.io 能够正常访问。
  2. 要有一个自己的域名，作者域名在 DNSPod 购买，以 DNSPod 面板为例。

# 申请证书

为了让我们的网站支持 HTTPS 访问，我们需要申请一个 HTTPS 证书。

这里我们使用的是 Freessl 提供的免费 SSL 服务，进去先注册登录，这样申请后的ssl证书可以保存在账号里。

首页输入自己的域名，根据使用需求选择证书类型，以 TRUST Asia 为例，在输入申请域名后，点击「创建」按钮，进入下一步。

输入签发证书的接收邮箱，自行选择证书的类型、验证方式和CSR文件生成方式后，点击创建。

以 浏览器生成 为例，保存好生成后下载的私钥文件。点击创建后，需要验证相关域名的 DNS 信息。

注意

不要点「点击验证」！你需要去你购买域名的地方，添加记录配置 TXT 类型，记录值是生成的。

如果你肯定自己填对了，「配置完成，检测一下」这一步不需要走。添加记录后坐等10分钟来点「点击验证」即可通过。

点击验证通过后，就会有证书生成，对应填入Gitee Pages Pro的配置证书选项。

点击提交，这样 HTTPS 证书就配置好了。

# 解析域名

提示

回到购买域名的 解析管理，将域名 CNAME 成功解析到 gitee.gitee.io 后，即可通过域名访问网站。

我这里添加了两条：

  1. www：填这个就解析www，你可以用 www.godlu.top 来访问。
  2. @：填这个就解析主域名，你可以用 godlu.top 来访问。

填写好后回到 Gitee Pages 服务这里，根据自己实际来填，有些是 master 分支。

  • 部署分支：用于部署 Pages 的分支。 # 选择你的分支
  • 部署目录：自定义 Pages 源文件的目录。 # 没有可以不填
  • 自定义域名：需解析 CNAME 到 gitee.gitee.io 生效后使用。 # www. 不填也不影响
  • 配置域名证书：用于配置自定义域名 HTTPS 证书。 # 上面已经说过了
  • 启动/暂停：启动/暂停 Gitee Pages 服务。 # 配好点击更新就行

# 总结

通过 申请证书->域名解析->部署 Gitee Pages 三个步骤，我们用 Gitee Pages 部署一个支持 HTTPS 的静态网站。除此之外，通过选择 Jekyll、Hugo 等不同静态网站生成工具和主题，就可以轻松打造出个人站点了。
