# Git

# 安装配置

# Windows

安装包下载地址：https://gitforwindows.org/

完成安装之后，就可以使用命令行的 Git 工具（已经自带了 ssh 客户端）

在开始菜单里找到"Git"，内有3个工具 Git Bash / Git CMD / Git GUI

# Git配置

Git 提供了一个叫做 git config 的工具，专门用来配置或读取相应的工作环境变量。

这些环境变量，决定了 Git 在各个环节的具体工作方式和行为。这些变量可以存放在以下三个不同的地方：

  • /etc/gitconfig 文件：系统中对所有用户都普遍适用的配置。若使用 git config 时用 --system 选项，读写的就是这个文件。
  • ~/.gitconfig 文件：用户目录下的配置文件只适用于该用户。若使用 git config 时用 --global 选项，读写的就是这个文件。
  • 当前项目的 Git 目录中的配置文件（也就是工作目录中的 .git/config 文件）：这里的配置仅仅针对当前项目有效。每一个级别的配置都会覆盖上层的相同配置，所以 .git/config 里的配置会覆盖 /etc/gitconfig 中的同名变量。

在 Windows 系统上，Git 会找寻用户主目录下的 .gitconfig 文件。主目录即 $HOME 变量指定的目录，一般都是 C:\Documents and Settings\$USER

此外，Git 还会尝试找寻 /etc/gitconfig 文件，只不过看当初 Git 装在什么目录，就以此作为根目录来定位。

# 用户信息

配置一个人的用户名称和电子邮件地址：

$ git config --global user.name "pangxi"
$ git config --global user.email xxxx@qq.com

检查已有的配置信息

$ git config --list
user.name=pangxi
user.email=xxxxx@qq.com

若要在某个特定的项目中使用其他名字或者电邮，只要去掉 --global 选项重新配置即可，新的设定保存在当前项目的 .git/config 文件里。

# 文本编辑器

设置Git默认使用的文本编辑器, 一般是 Vi 或者 Vim 或 Emacs。

$ git config --global core.editor <编译器名>

# 流程概念

# 工作流程

  • 克隆 Git 资源作为本地工作目录
  • 在克隆的资源上添加或修改文件。
  • 如果其他人修改了，你可以更新资源。
  • 在提交前查看修改。
  • 提交修改。
  • 如果发现错误，可撤回提交并再次修改并提交。

# 工作区、暂存区、版本库

  • 工作区：就是你电脑里的本地目录。
  • 暂存区：英文叫stage或index。一般存放在 .git/index ，所以暂存区有时也叫索引（index）。
  • 版本库：工作区里有个隐藏目录 .git ，这个不算工作区，而是Git的版本库。

# 创建与获取项目

# git init

git init 在目录中创建新的 Git 仓库。你可以在任何时候、任何目录中这么做，完全是本地化的。

# 初始化当前目录作为仓库
$ git init

# 新建目录作为仓库
$ git init <目录名>

以上两个命令都会生成 .git 目录。

若文件纳入版本控制，先用 git add 命令告诉 Git 对这些文件进行跟踪，然后提交：

$ git add *.c # 目录下以 .c 结尾的文件
$ git add README # README文件
$ git commit -m '提交说明：好好好'

# git clone

我们使用 git clone 从现有 Git 仓库中拷贝项目

# 克隆仓库的命令格式；若克隆到指定的新目录，命令末尾指定新的名字。
$ git clone <repo> <directory>
$ git clone git://github.com/schacon/grit.git

# 基本快照

Git 的工作就是创建和保存你的项目的快照及与之后的快照进行对比。

# git add

git add 可将该文件添加到缓存。

# 当前目录添加到缓存
$ git add .

# 对文件名添加到缓存
$ git add <文件名>

# git status

git status 查看在你上次提交之后是否有修改。

# 以精简的方式显示文件状态
$ git status -s

# git diff

git diff 查看执行 git status 的结果的详细信息。

# 查看未缓存的改动
$ git diff

# 查看已缓存的改动
$ git diff --cached

# 查看为缓和已缓的改动
$ git diff HEAD

# 显示摘要而非整个diff
$ git diff --stat

# git commit

git add 将想要快照的内容写入缓存区，而 git commit 将缓存区内容添加到仓库中。

# -a以跳过 git add 提交缓存的流程
$ git commit -a

# -m以在命令行中提供提交注释：
$ git commit -m '你的描述'

# 两者兼顾
$ git commit -am '你的描述'

# git reset HEAD

用于取消已缓存的内容，以取消之前 git add 添加，但不希望包含在下一提交快照中的缓存。

$ git reset HEAD <file>

# git rm

要从 Git 中移除某个文件，就要从已跟踪文件清单中移除;-f。

# 删除文件，从已跟踪文件清单中移除
$ git rm <file>

# 如果删除之前修改过并且已经放到暂存区域的话，要用强制删除选项
$ git rm -f <file>

# 不从工作区中删除文件，仅是从跟踪清单中删除
$ git rm --cached <file>

# 递归删除整个目录中的所有子目录和文件
git rm –r *

# git mv

git mv 命令用于移动或重命名一个文件、目录、软连接。

# 对其重名
$ git mv README  README.md

# 分支管理

有人把 Git 的分支模型称为必杀技特性，而正是因为它，将 Git 从版本控制系统家族里区分出来。

# 创建、切换、列出、删除、合并

# 创建分支
$ git branch <branchname>

# 切换分支
$ git checkout <branchname>
$ git checkout -b <branchname> # 创建分支并切换到该分支

# 列出分支
$ git branch

# 删除分支
$ git branch -d <branchname>

# 合并分支
$ git merge <分支名>

# 合并例子

$ git branch # 先用 branch 查看分支
* master
  newtest
$ ls
README        test.txt
$ git merge newtest # 合并分支
$ git branch -d newtest # 合并完后就可以删除分支:

# 合并冲突

合并并不仅仅是简单的文件添加、移除的操作，Git 也会合并修改。有点难理解。

# 查看提交历史

# git log

# 查看回顾下提交历史
$ git log

# --oneline

# 查看历史记录的简洁版本
$ git log --oneline

# --graph

# 开启了拓扑图选项：可以更清楚地看到何时工作分叉、又何时合并
$ git log --graph

# --reverse

# 逆向显示所有日志
$ git log --oneline --reverse

# --author=<用户名>

# 查找指定用户的提交日志
$ git log --author=Linus --oneline -5

# 指定日期

# 可以执行几个选项：--since 和 --before，也可以用 --until 和 --after
$ git log --oneline --before={3.weeks.ago} --after={2010-04-18} --no-merges
